# Logger Module: Context & Rules

Этот модуль обеспечивает структурированное логирование с поддержкой `rich`.

## 1. Архитектура

- **core.py**: Класс `ChutilsLogger`, расширяющий стандартный `logging.Logger`.
- **masking.py**: Логика автоматической маскировки чувствительных данных.
- **handlers.py**: Кастомные обработчики (SafeTimedRotatingFileHandler).
- **formatters.py**: Форматирование для консоли и JSON.

## 2. Важные правила

- **Инициализация**: Используйте `setup_logger()`. Она автоматически настраивает обработчики согласно конфигу.
- **Маскировка**: Список полей для маскировки настраивается через `Logging.mask_fields` в `config.yml`.
- **Rich**: По умолчанию используется `RichHandler` для красивого вывода. Не используйте `print()` внутри модуля.

## 3. Contextual Logging

Всегда используйте `chutils.bind_context` для добавления метаданных к логам внутри блока кода. Эти данные автоматически
попадут во все сообщения лога в текущем потоке/контексте.
