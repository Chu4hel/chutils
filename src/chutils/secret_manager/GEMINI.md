# Secret Manager Module: Context & Rules

Этот модуль обеспечивает безопасное управление секретами через различные провайдеры.

## 1. Архитектура

- **core.py**: Основной класс `SecretManager`.
- **providers.py**: Провайдеры (Keyring, Environment, Cloud).

## 2. Важные правила

- **Keyring**: Основной провайдер по умолчанию. Требует установленной библиотеки `keyring` и системного бэкенда.
- **Service Name**: Всегда указывайте осмысленное имя сервиса. Оно используется как пространство имен в системном
  хранилище.
- **Fallback**: Если секрет не найден в основном хранилище, `SecretManager` может проверять переменные окружения.

## 3. Безопасность

Никогда не логируйте значения, полученные из `SecretManager`. Хотя логгер пытается их маскировать, лучше избегать
прямого вывода секретов в лог.
