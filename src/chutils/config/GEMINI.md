# Config Module: Context & Rules

Этот модуль отвечает за загрузку, парсинг и мониторинг конфигурации приложения.

## 1. Архитектура

- **manager.py**: Синглтон `ConfigManager`, управляющий состоянием.
- **providers.py**: Провайдеры данных (YAML, INI, Env, Local).
- **watcher.py**: Мониторинг изменений файлов на базе `watchdog`.
- **getters.py**: Типизированные обертки для доступа к данным.

## 2. Важные правила

- **Приоритеты**: Environment Variables > `config.local.yml` > `config.yml`.
- **Hot-Reload**: Не кэшируйте значения конфига в локальных переменных модулей. Всегда используйте `get_config_value`
  или подпишитесь на изменения через `on_config_change`.
- **Пути**: Все пути в конфиге автоматически разрешаются относительно корня проекта, если используется
  `get_config_path`.

## 3. Расширение

Для добавления нового формата конфига реализуйте наследника `BaseConfigProvider` в `providers.py` и зарегистрируйте его
в `ConfigManager`.
